# Autopilot session — things only Reiss can do

This file collects every manual step needed after the autopilot run.
I log each item as I hit it so you have one checklist when you're back.

## Env vars to set (Netlify + `.env.local`)

### Linear (OAuth)
- `LINEAR_CLIENT_ID=...`
- `LINEAR_CLIENT_SECRET=...`

### Todoist (OAuth)
- `TODOIST_CLIENT_ID=...`
- `TODOIST_CLIENT_SECRET=...`

### Resend (API key — per-user, not env)
- _No env var needed._ Each user pastes their own key via the console modal.

### Plaid (server-side API)
- `PLAID_CLIENT_ID=...`
- `PLAID_SECRET=...`
- `PLAID_ENV=production` (or `sandbox` / `development`)

### Google Drive (reuses existing Google client)
- Reuses `GOOGLE_CLIENT_ID` and `GOOGLE_CLIENT_SECRET` (same as Gmail)
- Add redirect URI in Google Cloud Console (see below)

### Sentry (optional — forwards errors from self-hosted `error_events`)
- `SENTRY_DSN=https://<publicKey>@<host>/<projectId>` (optional — if unset, errors only land in Supabase)
- `SENTRY_ENV=production` (optional, defaults to NODE_ENV)
- `SENTRY_RELEASE=<git-sha>` (optional)
- No SDK installed — uses raw envelope POST to keep bundle slim.

### PostHog (optional — forwards analytics from self-hosted `analytics_events`)
- `POSTHOG_KEY=phc_...` (optional — if unset, events only land in Supabase)
- `POSTHOG_HOST=https://eu.i.posthog.com` (optional, defaults to EU cloud)
- No SDK installed — uses raw `/capture/` HTTP endpoint.

### Slack (slash command + events API)
- `SLACK_CLIENT_ID=...`, `SLACK_CLIENT_SECRET=...` (already required for OAuth — see messaging framework)
- `SLACK_SIGNING_SECRET=...` (NEW — required to verify `/api/slack/command` + `/api/slack/events` webhook signatures)

At https://api.slack.com/apps → your JARVIS app:
- **Slash Commands** → add `/jarvis`
  - Request URL: `https://<your-domain>/api/slack/command`
  - Short description: "Ask JARVIS anything"
  - Usage hint: `[command or question]`
- **Event Subscriptions** → enable, Request URL: `https://<your-domain>/api/slack/events`
  - Subscribe to bot events: `app_mention`, `message.im`
- **OAuth & Permissions** → Bot Token Scopes (add to existing):
  - `commands`, `app_mentions:read`, `im:history`, `im:read`, `im:write`, `chat:write`
- Reinstall the app to the workspace so the new scopes take effect.

### Telegram (shared bot + per-user link codes)
- `TELEGRAM_BOT_TOKEN=...` (REQUIRED — get from @BotFather)
- `TELEGRAM_BOT_USERNAME=...` (REQUIRED — e.g. `JarvisAssistantBot`, no @ sign)
- `TELEGRAM_WEBHOOK_SECRET=<random>` (REQUIRED — any random string, also passed to setWebhook)

Setup:
1. Talk to @BotFather on Telegram → `/newbot` → pick name + username → copy token.
2. Deploy JARVIS with the env vars above.
3. Register the webhook (one-off curl):
   ```
   curl "https://api.telegram.org/bot<TOKEN>/setWebhook" \
     -d url=https://<your-domain>/api/telegram/webhook \
     -d secret_token=<TELEGRAM_WEBHOOK_SECRET>
   ```
4. (Optional) `@BotFather → /mybots → <your-bot> → Edit Commands` to register:
   - `start - Link your JARVIS account (usage: /start <code>)`
   - `unlink - Remove the link between this chat and JARVIS`
   - `help - How to use the bot`

## OAuth apps to register at provider dashboards

### Linear
- Go to https://linear.app/settings/api/applications
- Create new OAuth2 application
- Redirect URL: `https://<your-domain>/api/integrations/linear/callback` (+ `http://localhost:3000/api/integrations/linear/callback` for dev)
- Scopes: `read`, `write`
- Copy client ID + secret into env vars above

### Todoist
- Go to https://developer.todoist.com/appconsole.html
- Create new app
- OAuth redirect URL: `https://<your-domain>/api/integrations/todoist/callback` (+ local)
- Scopes are requested per-flow; nothing to configure here

### Plaid
- Go to https://dashboard.plaid.com
- Grab client_id + secret for the environment (sandbox / development / production)
- Add allowed redirect URIs at Team Settings → API → Allowed redirect URIs:
  - `https://<your-domain>/api/integrations/plaid/callback`
  - `http://localhost:3000/api/integrations/plaid/callback`
- Products to enable: `transactions`, `auth` (auth optional but useful later)
- Country codes enabled: US, GB, CA, IE, FR, ES, NL, DE (already passed in `/link-token` route)

### Google Drive
- In Google Cloud Console → OAuth 2.0 Client IDs → the existing "JARVIS" client
- Add Authorized redirect URI:
  - `https://<your-domain>/api/integrations/drive/callback`
  - `http://localhost:3000/api/integrations/drive/callback`
- Enable Google Drive API in APIs & Services → Library
- Add scope `.../auth/drive` to the OAuth consent screen's scope list

### Resend
- Each end-user generates their own API key at https://resend.com/api-keys
- They paste it into the "Connect Resend" modal in the integrations console
- They also need a verified domain (resend.com/domains) if they want a default-from sender
